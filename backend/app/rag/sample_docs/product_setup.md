# Product Setup

Category: technical
Applies to: Initial product setup, installation, workspace configuration, and team onboarding
Policy: Support may provide configuration guidance when the request does not require privileged account changes or access to customer secrets.
Steps:
1. Confirm the product area and the customer's intended setup.
2. Share the documented installation or configuration sequence.
3. Ask for the exact error message if a setup step fails.
4. Escalate requests that require privileged access or expose credentials.
Escalation rules: Escalate security-sensitive configuration, credential exposure, or requests for direct access to customer systems.
Keywords: product, setup, configure, install, onboarding, workspace, team, support
